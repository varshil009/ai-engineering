📚 How SSE Streaming Works

📦 Accumulate & Flush

The LLM tokens are accumulated server-side into a buffer. When the buffer reaches a threshold (e.g., 5 characters), the batch is flushed to the client as a single SSE event. This simulates a more "chunked" typing feel and reduces network overhead.

Buffer → threshold met → flush → SSE event → client renders

⚡ Direct Yield

Every token generated by the LLM is immediately yielded to the client one-by-one via SSE. This provides true real-time streaming with minimal latency, but at the cost of more network events.

Token → immediately yield → SSE event → client renders
